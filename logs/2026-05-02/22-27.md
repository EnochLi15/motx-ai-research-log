# Stateful Learning Loop 与系统级自进化机制

时间：2026-05-02 22:27

## 本轮新增发现

前几轮已经建立完整控制闭环：

```text
Intent → Plan → Contract → Identity → Permission → Sandbox → Execution → Verification → Evidence → Decision
```

但存在一个关键缺失：

> **系统没有“长期记忆”和“策略进化能力”，每一次决策都是局部最优，而不是全局持续优化。**

当前问题：

```text
- Decision 是一次性的，没有系统性反馈
- Agent 表现没有长期结构化积累
- Verification Pipeline 不会自动优化
- Contract 不会根据失败演化
- 不同 repo / 团队之间经验无法迁移
```

本轮关键新增：

> **Stateful Learning Loop（有状态学习闭环） + System-level Evolution Engine**

## MOTX/Moxt 产品理解更新

## AI 同事角色设计更新

新增两个系统级 AI：

### 1. Outcome Analyst / 结果分析 AI

**定位**：分析每一次 ReleaseDecision 之后的真实结果（Outcome），评估决策质量。

#### 职责

- 收集 Outcome：是否引入 bug、是否被 rollback、是否产生 incident、性能变化；
- 计算 decision correctness 与 risk prediction accuracy；
- 标注 false accept、false reject；
- 输出 structured feedback。

#### 工具权限

- 可读取 production metrics、incident logs、monitoring systems；
- 不可修改代码；
- 不可修改决策。

#### 评价指标

```text
outcome coverage rate
incident detection latency
label accuracy
feedback usefulness
```

### 2. Evolution Engine / 策略进化引擎

**定位**：根据历史 Outcome 和系统行为，优化 DecisionPolicy、VerificationPipeline、ControlContract、Agent routing。

#### 职责

- 分析历史数据：DecisionTrace、Outcome、AgentPerformance；
- 识别模式：哪类任务容易失败、哪些 agent 更可靠；
- 更新策略：调整 acceptance threshold、强化某些 verification steps、收紧或放宽 contract；
- 输出 policy updates、pipeline updates、contract updates。

#### 工具权限

- 可写 DecisionPolicy（受控）、Pipeline templates、Contract templates；
- 不可直接发布代码；
- 高风险更新必须经过 human approval。

#### 评价指标

```text
system improvement rate
incident reduction rate
decision accuracy improvement
policy stability
regression rate
```

## 软件研发新作业范式更新

范式升级为：

> **Self-Evolving Agentic Development System**

核心流程扩展：

```text
1. Intent → Plan → Contract
2. Execution（Agent）
3. Verification → Evidence
4. Decision
5. Release
6. Outcome（真实世界）
7. LearningSignal
8. Evolution（策略更新）
```

### 核心变化

#### 1. 系统具备长期记忆

不再每次任务独立，而是所有任务共享经验。

#### 2. 决策可持续优化

```text
Decision(t) → Outcome(t) → Improve Decision(t+1)
```

#### 3. Pipeline / Contract 自动进化

系统会强化有效规则，淘汰无效规则。

## 效率提升机制

### 1. 持续降低错误率

通过 false accept → policy tightening。

### 2. 持续降低成本

通过 low-risk + high success → 减少 verification。

### 3. 自动优化 agent 使用

系统会学会哪个 agent 适合哪类任务。

### 4. 减少人为调参

过去工程团队手动调 CI / review / policy，现在系统自动优化。

## 风险与边界

### 1. 反馈延迟问题

Outcome 可能在数天后才出现，导致学习滞后。

### 2. 错误学习（Mislearning）

如果数据偏差，系统可能学到错误策略。

### 3. 策略震荡（Policy Oscillation）

过度调整会导致系统不稳定。

### 4. 复杂度爆炸

系统包含 policy + pipeline + contract + decision + learning，需要强抽象。

## 阶段性结论

Moxt 系统最终演化为：

```text
Contract
```
