# Moxt 进入可学习编排系统阶段（Learning Orchestrator）

时间：2026-05-02 14:28

## 一、本轮新增发现（关键突破）

### 1. 系统瓶颈再次迁移：从调度到学习能力

上一轮我们确认：核心在 Orchestration（调度），本质是 Solution Space 搜索。本轮新增关键判断：

> 如果系统不能“记住哪种策略有效”，调度将无法持续优化。

即：

```text
调度（Orchestration） → 学习（Learning） → 再调度
```

Moxt 若要形成护城河，必须具备跨任务的策略学习能力（cross-task learning）。

### 2. 引入核心新机制：Strategy Memory（策略记忆）

区别于之前的 Fact Memory、Policy Memory、Outcome Memory，本轮新增 Strategy Memory。

它存储：

```text
在什么任务类型 + 什么约束下 → 哪种调度策略最优
```

示例：

```yaml
strategy_case:
  task_type: API refactor
  repo_size: large
  risk_level: medium
  best_strategy:
    parallel_agents: 2
    agent_types:
      - fast_model
      - code_specialist
    early_stop: true
```

这是系统从“执行系统”进化为“经验系统”的关键。

### 3. 研发系统进入 RL-like（强化学习式）优化阶段

重要类比：

> Moxt 的长期形态近似一个强化学习系统。

| RL 概念 | Moxt 对应 |
|---|---|
| State | Task + Context |
| Action | 调度策略 |
| Reward | 质量 / 成本 / 时间 |
| Policy | Orchestrator 行为 |

关键推论：Orchestrator AI 本质是一个 policy learner。

### 4. 从单任务最优到全局效率最优

过去每个任务独立优化；现在应优化整个研发系统的吞吐 / 成本 / 质量平衡。例如：某些任务用 cheap agent，把预算留给关键路径任务。

## 二、Moxt 产品理解更新

### 从 Orchestration System 到 Learning Orchestrator

升级定义：

> Moxt = Learning-based AI Engineering Orchestration System。

新增核心能力：

1. **Policy Learning（策略学习）**：自动学习最优调度方式。
2. **Cross-task Optimization（跨任务优化）**：不是每个任务独立决策。
3. **Budget Allocation Intelligence（预算分配智能）**：AI work / credits 作为资源进行最优分配。

### 更新后的系统结构

```text
1. Intent Layer
2. Constraint Layer
3. Orchestration Layer
4. Execution Layer
5. Verification Layer
6. Learning Layer（新增核心）
```

本轮核心：Learning Layer 成为闭环关键。

## 三、AI 同事角色设计：新增学习型角色

### Learning Controller（学习控制者）

定位：负责让系统“越来越聪明”，而不是完成单次任务。

#### 核心职责

1. 收集每次任务的执行数据、agent 表现、选择结果。
2. 分析哪种策略有效、哪种 agent 在何种任务表现好。
3. 更新 Strategy Memory、Agent ranking、调度策略。

#### 工具权限

- 读全量任务数据。
- 修改 strategy memory。
- 调整 agent 权重。

#### 协作边界

- 不参与具体任务执行。
- 不直接影响单次任务，而是通过策略间接影响。

#### 评价指标

- 系统整体效率提升率。
- 平均成本下降幅度。
- 成功率提升。
- 学习收敛速度。

关键升级：AI 同事从“执行角色”扩展到“系统进化角色”。

## 五、新作业范式

从搜索解空间，升级为：

> 持续优化解空间搜索策略。

### 新范式

```text
Solve → Evaluate → Learn → Adapt → Solve Better
```

本质变化：研发系统变成“自优化系统”。

## 六、效率提升机制

### 1. 经验复用

过去每个任务从零开始；现在使用历史最优策略。

### 2. Agent 专业化

系统自动学会哪个 agent 擅长什么，并自动分配。

### 3. 成本-质量动态平衡

系统学会什么任务值得花钱，什么任务用 cheap agent。

### 4. 全局最优而非局部最优

系统优化整体 throughput，而不是单任务完美。

## 七、风险与边界

### 1. 学习错误（Wrong Generalization）

系统学错规律，会持续做错误决策。

### 2. 探索不足（Under-exploration）

过早收敛会错过更优策略。

### 3. 数据偏差（Bias）

训练数据不均会导致策略偏向某类任务。

### 4. 黑箱决策（不可解释）

系统变复杂后，人无法理解为何这么调度。

## 八、阶段性结论

最终系统模型升级为：

> Moxt = 一个可学习的 AI 软件研发操作系统。

其核心能力三层：

```text
1. Orchestrate（编排）
2. Evaluate（评估）
3. Learn（学习）
```

最关键变化：系统价值从“执行能力”转移到“学习能力”。

## 九、下一轮研究方向

1. 设计 Reward Function（奖励函数）：如何定义“好”的统一指标。
2. 构建 Exploration Strategy：如何平衡尝试 vs 利用。
3. 设计 Multi-Objective Optimization：质量 vs 成本 vs 时间。
4. 推导 Human Override 最优机制：人类如何干预学习过程。
5. 深入 Strategy Memory 数据结构设计：如何高效存储与检索策略。
