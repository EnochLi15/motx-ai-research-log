# 2026-05-03 22:27 — Self-Evolving System Layer

## 本轮新增发现
规则、约束和决策图谱不会自动进化，系统必须具备反馈闭环自动化。新增 Self-Evolving System Layer：从错误中提取模式，从决策中提取最佳实践，自动生成 Rule Candidates、Constraint Candidates、Skill Candidates。

## MOTX/Moxt 产品理解更新
Moxt 从可执行、可控制升级为可进化。结构新增自进化层，负责把 Incident、Pattern、Candidate、Validation 串成学习闭环。

## AI 同事角色设计
新增 AI Learning & Memory Curator，分析 Bug、回滚、Review 争议、测试缺失，提取重复模式并生成候选规则或约束。它只能写入 Memory Candidates，不能直接修改正式 Rules / Constraints。

## AI 原生工作空间设计
新增 Learning Loop Layer，包含 Incidents、Patterns、Candidates、Validation。关键链路为 Incident → Pattern → Candidate → Validation → Rule/Constraint。

## 软件研发新作业范式
Human defines intent & boundaries → AI executes → System enforces → Decisions recorded → System learns → Constraints evolve。

## 效率提升机制
新增 Learning Compounding：同类 Bug 只发生一次，Review 评论减少，测试覆盖自动增强，架构错误逐渐消失。

## 风险与边界
错误学习、规则爆炸、系统僵化。需要验证阶段、多案例支持、人类抽查、Rule Pruning 和实验空间。

## 阶段性结论
Moxt 终局不是 AI 帮你写代码，而是构建一个会变得越来越强的研发组织系统。

## 下一轮研究方向
Learning Loop 最小实现、Rule Lifecycle、规则冲突解决、从 Decision Graph 自动生成测试用例。
